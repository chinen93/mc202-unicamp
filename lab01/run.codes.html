O problema do acesso à lista

Disciplina: MC202 - Estrutura de Dados
Descrição:
Prazo máximo de submissão: 29/08/2015

O problema do acesso à lista é um dos problemas computacionais de maior significado histórico, teórico e de interesse prático. Ele é aparentemente um problema muito simples, mas esconde várias sutilezas (veja o verbete na Wikipedia).

Imagine que temos um fichário contendo n pastas rotuladas mas sem nenhuma ordem especial (uma lista). Cada rótulo é uma chave (inteiro) que serve para identificar a pasta de modo único. O objetivo é atender a uma sequência de requisições onde uma requisição é uma das seguintes operações: acesse(x), insere(x) e remove(x), onde x é um rótulo de um item (pasta). O custo de uma requisição x é 1 mais o número de itens que precedem x (1 mais o tamanho da lista se o item não pertence à lista). Mais precisamente, acessar ou remover o i-ésimo item da lista custa i/ Inserir um novo item custa n+1 onde n é o tamanho da lista (o item é inserido no final da lista).

Você como o responsável pelo fichário, gostaria de usar uma estratégia que permitisse minimizar o tempo total que você gasta para atender às requisições. Isto talvez fosse mais fácil se você soubesse de antemão quais as requisições futuras, mas no contexto explicado acima o problema é online (informalmente, significa que as requisições vão chegando e devem ser atendidas imediatamente sem conhecimento do futuro).

É fácil perceber que uma lista ligada é a estrutura de dados mais adequada para implementar o modelo acima. O custo das buscas é proporcional ao número de itens percorridos.

Dois algoritmos para acesso à lista

Nos algoritmos de acesso à lista costuma-se adotar a seguinte convenção. Imediatamente após acessar ou inserir um item, podemos movê-lo (sem nenhum custo) para qualquer posição próxima ao início da lista. As transposições (trocas) feitas para realizar este movimento são chamadas livres enquanto as demais transposições são chamadas pagas e tem custo 1 cada.

Muitos algoritmos foram propostos para gerenciar uma lista. Neste projeto consideramos dois algoritmos bem conhecidos que só usam transposições livres.

  • MTF (Move-To-Front): após acessar um item, este é movido para o início da lista;
  • TR (Transpose) : após acessar ou inserir um item, faça uma transposição com o item anterior (sem custo). Se o item acessado for o primeiro da lista, mantenha-o na mesma posição;

Vejamos um exemplo concreto. Denote as operações de acesso, inserção e remoção por a(x), i(x) e r(x), respectivamente. Suponha que a lista inicial seja L=(1,2,3,4,5) e a sequência de requisições sejam a(4),a(2),a(2),i(6),a(3),r(2),a(3),a(6),r(1). Veja abaixo quais são os custos para cada algoritmo.

MTF

  • Lista inicial L=(1,2,3,4,5)
  • a(4). Custo = 4. Lista L=(4,1,2,3,5)
  • a(2). Custo = 3. Lista L=(2,4,1,3,5)
  • a(2). Custo = 1. Lista L=(2,4,1,3,5)
  • i(6). Custo = 6. Lista L=(2,4,1,3,5,6)
  • a(3). Custo = 4. Lista L=(3,2,4,1,5,6)
  • r(2). Custo = 2. Lista L=(3,4,1,5,6)
  • a(3). Custo = 1. Lista L=(3,4,1,5,6)
  • a(6). Custo = 5. Lista L=(6,3,4,1,5)
  • r(1). Custo = 4. Lista L=(6,3,4,5)
Custo total = 4+3+1+6+4+2+1+5+4 = 30.

TR

  • Lista inicial L=(1,2,3,4,5)
  • a(4). Custo = 4. Lista L=(1,2,4,3,5)
  • a(2). Custo = 2. Lista L=(2,1,4,3,5)
  • a(2). Custo = 1. Lista L=(2,1,4,3,5)
  • i(6). Custo = 6. Lista L=(2,1,4,3,6,5)
  • a(3). Custo = 4. Lista L=(2,1,3,4,6,5)
  • r(2). Custo = 1. Lista L=(1,3,4,6,5)
  • a(3). Custo = 2. Lista L=(3,1,4,6,5)
  • a(6). Custo = 4. Lista L=(3,1,6,4,5)
  • r(1). Custo = 2. Lista L=(3,6,4,5)
Custo total = 4+2+1+6+4+1+2+4+2 = 26.

Resultados conhecidos

Apesar do exemplo acima, o TR é o pior dos dois algoritmos, no sentido de que para a maiorias das instâncias, o MFT se comporta melhor que o TR. Pode-se provar isto formalmente.

Especificação da entrada e saída

Você pode supor que as chaves são inteiros entre 1 e 1000.

A primeira linha da entrada contém dois inteiros N e R, representando o tamanho inicial da lista e o número de requisições, com 1 ≤ N ≤ 100 e 1 ≤ R ≤ 200. A segunda linha contém N inteiros distintos representando os itens da lista. Segue-se então R linhas cada uma contendo uma das seguintes instruções:

  • a <x> (acesse item x)
  • i <x> (insira item x)
  • r <x> (remova item x)
onde <x> é um inteiro. 
Entrada:
5 9
1 2 3 4 5
a 4
a 2
a 2
i 6
a 3
r 2
a 3
a 6
r 1

A saída do programa consiste de 4 linhas contendo o custo de MTF, a lista final, o custo de TR e a lista final quando os algoritmos são executados para a entrada dada.

Saída:

30
6 3 4 5
26
3 6 4 5

Observações

  • Suponha que cada linha da entrada ou da saída termina com um `\n'.
  • O número máximo de submissões é 15.
  • Há várias maneiras de fazer o projeto e algumas nem precisam usar listas ligadas. Entretanto, como o objetivo do projeto é se familiarizar com listas ligadas, você deve implementar os algoritmos MTF e TR sobre uma lista ligada. Você pode usar listas ligadas com nó cabeça, se quiser. Se quiser, pode usar listas duplamente ligadas.
  • Se o programa não passar em todos os testes, a nota final será no máximo 4. O código submetido será lido e poderão ser descontados pontos por falta de comentários e indentação, por implementação incorreta ou muito ineficiente das funções, ou por outros problemas de apresentação do código.
  • Casos de plágio acarretam média final zero para todos os envolvidos, sem exceção. Então não empreste trechos de código, não copie trechos de código, faça você mesmo o programa etc.
