Déjà vu


A MedievalSoft, que é  a mais nova responsável por implementar o sistema de controle de motoristas de patinete de todo o país, continua em seu projeto para desenvolver um sistema de cadastro de seus usuários.

Para tal,  a lista de pessoas será sempre armazenada num arquivo binário, o qual será composto de R registros (0<=R<=1000000), onde cada registro possui um identificador inteiro sem sinal (4 bytes) e uma sequencia de caracteres (1000 bytes) que são as informações do registro.

struct Record {
    unsigned int id;
    char data[1000];
}

Este software deverá ser capaz de realizar inserções e remoções destes, através do identificador contido em cada um deles. 

Já sabemos que buscar um identificador percorrendo todo o arquivo pode ser extremamente custoso. Por esse motivo, foi necessário adotar uma estrutura de busca rápida dos identificadores, onde decidiu-se por uma árvore de busca. Essa árvore será ordenada pelos identificadores dos registros, além de conter o endereço daquele registro no arquivo binário.

Devido a péssima experiência que alguns desenvolvedores tiveram ao tentar implementar uma árvore AVL, optou-se por utilizar uma nova estrutura para substituí-la: uma árvore-B.

Como a empresa também se recusa a utilizar debuggers e ambientes de desenvolvimento, precisaremos desenvolver um software para nos ajudar a checar o desbalanceamento dessas árvores. Este programa deverá, basicamente:
  1. Ler uma base de registros contida num arquivo, criando uma árvore-B para busca rápida dos registros;
  2. Realizar uma série de operações de inserção e remoção, neste arquivo;
  3. Imprimir um conjunto solicitado de nós da árvore, juntamente com seu endereço correspondente no arquivo.


Objetivo (e dicas mais objetivas)

A primeira tarefa será ler todo o arquivo binário, criando uma árvore AVL ordenada de acordo com os identificadores de cada registro. Além desse identificador será necessário guardarmos a localização daquele registro no arquivo para que, futuramente, possamos acessá-lo ou alterá-lo.

 Logo em seguida teremos N operações (0<=N<=100000), que poderão representar casos de remoção ou inserção.

Seguiremos a seguinte sintaxe:

  • r <identificador>
    • indica que o registro com o identificador indicado deverá ser removido. O processo de remoção consiste apenas de guardar aquele endereço (numa heap) do arquivo, que agora será considerado um endereço  vazio e disponível para posterior uso.
  • i <identificador> <string de dados>

    • indica que um registro com o identificador indicado deverá ser inserido (juntamente com sua string de dados) no primeiro endereço vazio do arquivo (lembrem-se que, nas heap eu posso encontrar o menor nó de forma rápida). Caso não existam endereços vazios deverá ser inserido imediatamente após o último registro.

    A próxima linha conterá um inteiro X (1<=X<=10), que representa uma altura da arvore.

    Por fim, a árvore deverá ser percorrida, imprimindo os dados dos nós que possuem altura igual a X (de forma a respeitar a ordem dos identificadores), além de sua localização no arquivo binário.


    Especificação de entrada e saída

    A primeira linha do problema indica o número máximo k de registros por página da árvore-B (3<=k<=100).

    Em seguida, uma string que indica o nome do arquivo binário onde temos os dados dos usuários, de acordo com a estrutura descrita acima.

    A linha seguinte contém um inteiro N com o número de ações a serem executadas.

    As N linhas seguintes terão ações de remoção ou inserção, do tipo:

    • r <identificador>
    • i <identificador> <string de dados>

    Por fim, a última linha terá um inteiro X (1<=X<=10) indicando a altura X dos nós a serem apresentados.

    A saída será composta de todos os nós (em ordem crescente de identificador).

    Cada nó será mostrado numa linha que contém a string de dados daquele nó (armazenada no arquivo) e o endereço daquele nó dentro do arquivo, separados por espaço.


    Exemplo de Entrada

    3

    database.bin
    6
    r 12
    i 12 A
    r 13
    i 13 B
    r 14
    i 14 C
    1

    Exemplo de Saída

    Raul Larson 3012

    Simone Wilderman 4016

    A 10040

    Observações

    Você deve implementar as TAD (Tipo Abstrato de Dados).

    Deverá ler e inserir as de strings dados do arquivo binário (ao invés de mantê-lo em memória).

    Deverá utilizar a árvore-B e Heap na implementação.

    Tentar condensar todo este problema em um texto de poucas linhas foi ainda mais difícil que implementá-lo, se houver qualquer ponto nebuloso, alerte seu PED para que possa corrigi-lo.
